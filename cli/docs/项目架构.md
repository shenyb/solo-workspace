# Solo Workspace 项目架构

> 开源 · 插件架构 · 开发者优先
> 独立开发者的命令行操作系统

---

## 一、项目概览

| 属性 | 值 |
|------|----|
| 语言 | Go 1.26+ |
| CLI 框架 | Cobra (`github.com/spf13/cobra`) |
| 配置格式 | YAML (`gopkg.in/yaml.v3`) |
| 加密 | AES-256-GCM (`golang.org/x/crypto`) |
| 输出渲染 | 自研 Table 渲染（支持 ANSI、自适应列宽） |
| TUI | Electron-like terminal UI（基于 Bubble Tea 风格） |
| 许可证 | MIT |

## 三、数据流

```
用户输入 sw <command> [args]
        │
        ▼
    cmd/root.go (Cobra)
        │
        ├── PersistentPreRun → LoadConfig()
        │       │
        │       ▼
        │   internal/config.go
        │   1. -c <path> 指定
        │   2. ~/.solo/config.yaml
        │   3. ./.solo.yaml
        │   4. 空默认值
        │
        ├── plugins/*/plugin.go
        │       │
        │       ▼
        │   读取/修改 CurrentConfig
        │       │
        │       ▼
        │   SaveConfig() → ~/.solo/config.yaml
        │
        └── internal/output.go
                │
                ▼
           终端输出（表格/JSON/颜色）
```

## 四、插件架构

每个插件是一个独立目录，遵循以下约定：

```
plugins/<name>/
└── plugin.go       # package <name>, func Cmd() *cobra.Command
```

注册方式：在 `cmd/root.go` 的 `init()` 中添加一行：

```go
rootCmd.AddCommand(<name>.Cmd())
```

## 五、配置系统

### 配置加载优先级

| 优先级 | 路径 | 用途 |
|--------|------|------|
| 1 | `-c <path>` / `--config <path>` | 手动指定 |
| 2 | `~/.solo/config.yaml` | 全局配置（跨项目） |
| 3 | `.solo.yaml`（当前目录） | 项目级配置 |
| 4 | _(无)_ | 空默认值 |

### 数据文件位置

数据文件（`env.yaml`、`secrets.enc`）与当前使用的配置文件同级目录存放。

- 默认 `~/.solo/config.yaml` → 数据在 `~/.solo/`
- 使用 `-c /path/to/config.yaml` → 数据跟随到 `/path/to/`

### Config 结构

```go
type Config struct {
    Servers  map[string]*ServerConfig  // 服务器
    Domains  []string                  // 域名
    Notify   *NotifyConfig             // 通知配置
    Projects map[string]*ProjectConfig // 项目
    Todos    map[string]*TodoConfig    // 待办
}
```

## 六、关键设计决策

### 1. 数据本地化
不做 SaaS、不依赖云服务，`~/.solo/config.yaml` 是唯一真相源。用户完全掌控数据。

### 2. 终端优先
所有操作都在终端完成，不依赖图形界面。`sw` 无参数直接展示概览。

### 3. 插件架构
每个功能是一个独立插件包，像乐高积木。三步添加新插件：创建目录 → 实现 Cobra 命令 → 注册到 root。

### 4. 自适应表格渲染
自研 Table 渲染替代 tablewriter，特点：
- ANSI 颜色码不计入列宽（`runewidth.StringWidth`）
- 中文等宽字符正确计算
- 超过终端宽度时等比压缩
- 截断时保留颜色并补充 reset 码

### 5. 文件级数据共享
env 和 secret 的数据目录跟随配置文件路径，与配置解耦。使用 `-c` 切换配置时数据和密钥自动跟随。
